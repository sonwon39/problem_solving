# /counterexample — 반례 생성 스킬

코드가 WA를 받았지만 사용자가 원인을 모를 때, 작은 반례를 만들어 제시합니다.

## 입력 파싱

- `/counterexample cpp/1149.cpp` — 파일 읽고 분석
- `/counterexample 1149` — 문제 번호로 파일 자동 찾기
- 코드 블록 직접 붙여넣기도 허용

## 분석 순서

### 1. 문제 명세 확인
- 입력 형식, 출력 형식, 제약 조건을 정확히 파악
- 필요하면 `https://www.acmicpc.net/problem/{번호}` WebFetch

### 2. 코드 의도 파악
- 사용자가 어떤 접근으로 풀었는지 읽어냄 (DP? 그리디? 시뮬레이션?)
- 접근 자체가 틀렸다면 반례를 만들 것도 없이 그 사실을 먼저 알림

### 3. 의심 구간 후보
다음 카테고리에서 반례 후보를 생성합니다.

**경계 조건**
- N=1, N=최소값, N=최대값
- 배열이 비어있거나 원소가 하나
- 모든 원소가 같은 경우
- 입력 한 줄짜리

**극단값**
- 0, 음수, 최대값
- 중복된 값
- 정렬되어 있는 / 역정렬된 입력

**구조적 엣지**
- 그래프: 사이클, 자기 루프, 중복 간선, 연결되지 않은 컴포넌트
- 문자열: 빈 문자열, 길이 1, 전부 같은 문자
- 트리: 일자형(체인), 별 모양

**자료형**
- int overflow 유발 입력 (N, M이 최대일 때 합/곱)
- 음수로 떨어지는 모듈러

### 4. 추적 & 검증
각 후보를 골라 **코드를 머리로 실행**하면서 예상 출력과 실제 예상 결과를 대조합니다.
차이가 나는 입력을 발견하면 그것이 반례입니다.

## 답변 형식

반례를 찾으면 이렇게 제시합니다.

```
반례를 찾았습니다.

입력:
3
1 2 3

예상 출력(정답): 6
현재 코드 출력: 5

추적:
- i=0에서 sum=1 ...
- 여기서 초기값이 0이 아니라 1로 시작해서 한 칸씩 밀림

힌트: 어느 줄에서 초기화를 잘못했는지 찾아보세요.
```

## 답변 규칙

1. **반례를 여러 개 남발하지 마세요.** 가장 작고 명확한 것 하나면 충분. 못 찾으면 그때 다른 걸 시도.

2. **반례 크기는 N ≤ 5** 를 우선. 손으로 추적 가능해야 학습 효과가 있음.

3. **원인을 바로 말하지 말고**, 반례와 기대 출력만 먼저 주고 사용자가 스스로 찾게 합니다. 사용자가 "못 찾겠어"라고 하면 그때 원인 설명.

4. **반례를 못 만들겠으면 솔직히 말합니다.** "제 분석으로는 반례를 못 찾았어요. 혹시 입력 파싱이나 출력 형식 문제일 수 있으니 확인해보세요."

5. 한국어로 답변.

## 금지 사항

- 반례 없이 "이 부분이 틀렸을 것 같아요" 추측만 말하기
- 거대한 입력(N=1000)을 반례로 제시하기
- 반례와 동시에 수정 코드까지 주기 (학습 방해)
