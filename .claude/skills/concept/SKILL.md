# /concept — 개념 설명 스킬

알고리즘/자료구조 개념을 백준 문제와 함께 설명합니다.

## 입력 파싱

- `/concept 세그먼트트리` — 개념 이름만
- `/concept 세그먼트트리 2042` — 개념 + 연관 BOJ 문제 번호
- `/concept dp` — 약어, 영문 모두 허용

## 설명 구조

항상 다음 5개 섹션으로 답변합니다. 섹션 순서를 지키세요.

### 1. 한 줄 요약
이 개념이 "무엇을 빠르게 하는가" 한 문장. 정의가 아니라 **효능** 중심.
예: "세그먼트 트리는 구간 합(또는 최솟값 등)을 O(log N)에 질의하고, 원소 하나를 O(log N)에 수정할 수 있게 해주는 자료구조입니다."

### 2. 언제 쓰는가
- 어떤 문제 상황에서 이 개념이 "정답"인지
- 어떤 신호(제약 조건, 연산 패턴)를 보면 이걸 떠올려야 하는지
- 대체 가능한 다른 방법과 비교 (예: 누적합 vs 세그트리 — 수정이 있으면 세그트리)

### 3. 직관 (Why)
- 왜 이 구조가 빠른지, 왜 이렇게 생겼는지를 **그림이나 작은 예제**로
- 가능하면 N=4~8 정도의 작은 트리/배열을 ASCII로 그려서 보여주기
- 증명은 하지 말고 "감"을 전달

### 4. 최소 예시 코드
- 핵심 연산(예: build, update, query) 하나씩만
- C++ / Python 중 사용자 선호 언어 (모르면 C++)
- 주석은 자료구조 내부 상태 변화 설명에 집중

### 5. 연습 문제 (난이도 순 3~5개)
- 가장 쉬운 것부터 점진적으로
- 각 문제에 한 줄로 "여기서 배우는 포인트"
- 백준 번호와 제목 같이

## 답변 규칙

1. **사용자의 문제 상황을 먼저 물어봅니다.** 특정 문제를 풀다가 개념을 물은 거라면, 그 문제 맥락으로 설명 톤을 맞춥니다.

2. **교과서식 나열 금지.** "세그먼트 트리의 종류: ① 합 ② 최솟값 ③ 최댓값..." 같은 식 말고, 하나의 흐름으로.

3. **코드는 적게, 직관은 많이.** 코드가 설명의 주인공이 되지 않도록.

4. **선수 개념 체크.** 세그트리를 물었는데 이진트리 자체가 낯설면 거슬러 올라가 먼저 설명.

5. 한국어로 답변.

## 자주 요청되는 개념 (빠른 참조)

- **자료구조**: 세그먼트 트리, 펜윅(BIT), 유니온 파인드, 트라이, 우선순위 큐, 덱, 스택
- **그래프**: BFS, DFS, 다익스트라, 벨만포드, 플로이드-워셜, 위상정렬, SCC, MST(크루스칼/프림), 이분 매칭
- **DP**: 1D/2D DP, 배낭, LIS, LCS, 구간 DP, 비트마스크 DP, 트리 DP, 자릿수 DP
- **탐색**: 이분탐색, 매개변수 탐색, 투 포인터, 슬라이딩 윈도우
- **문자열**: KMP, 라빈-카프, Z algorithm, 접미사 배열
- **수학**: 에라토스테네스, 모듈러 역원, 행렬 거듭제곱, 분할정복 거듭제곱
