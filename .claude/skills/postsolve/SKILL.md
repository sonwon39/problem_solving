# /postsolve — 풀이 후 회고 스킬

문제를 푼 직후, 단순히 "다음 문제로" 넘어가지 않고 학습을 정착시키는 단계입니다.

## 입력 파싱

- `/postsolve 1149` — 문제 번호. 해당 파일을 `cpp/1149.*` 또는 `python/1149.*`에서 찾음
- `/postsolve` — 직전 대화에서 풀이한 문제를 대상으로

코드를 못 찾으면 사용자에게 붙여넣어 달라고 요청합니다.

## 회고 항목

답변은 다음 5개 섹션으로 구성합니다.

### 1. 한 줄 핵심 아이디어
이 문제를 "다음에 비슷한 걸 봤을 때 즉시 떠올릴 수 있는" 한 문장으로 요약.
예: "1149번의 핵심은 '직전 선택에 의존하는 1D DP — 상태를 (위치, 직전 선택)으로 확장'입니다."

### 2. 문제 패턴 태그
다른 문제에서 이 패턴을 알아볼 수 있도록 태그 2~3개를 붙입니다.
예: `#1D-DP` `#상태확장` `#직전선택의존`

### 3. 내 코드에서 잘한 점 / 개선할 점
- 사용자의 실제 코드를 읽고 구체적으로
- 잘한 점도 반드시 언급 (긍정 강화)
- 개선점은 1~2개만, 가장 임팩트 있는 것

### 4. 놓치기 쉬운 포인트
- 이 유형의 문제에서 자주 실수하는 것
- 사용자가 이번에 안 걸렸지만 다음에 비슷한 문제에서 걸릴 수 있는 함정
- 예: "색 3개가 고정이지만, 색이 k개로 일반화되면 비트마스크 DP로 넘어가야 함"

### 5. 다음 문제 추천 (2~3개)
비슷한 패턴이지만 조금 더 어렵거나 변형이 있는 문제.
- 바로 다음: 같은 유형, 조금 더 어려움
- 응용: 같은 아이디어 + 다른 요소 결합
- 각 문제에 한 줄 이유

## 답변 규칙

1. **사용자 코드를 읽어보세요.** 추상적 조언이 아니라 "이 코드의 42번째 줄 방식은 좋았다/이렇게 바꾸면 더 나았을 것"처럼 구체적으로.

2. **긍정 먼저.** 학습 정착이 목표이므로 성취감을 해치지 않습니다.

3. **개선점은 짧게, 핵심만.** 이미 AC 받은 코드를 과하게 분해하지 마세요.

4. **다음 문제 추천은 난이도를 살짝만 올립니다.** 점프하면 학습 곡선이 끊어짐.

5. 한국어로 답변.

## 금지 사항

- "이 코드는 완벽합니다" 같은 내용 없는 칭찬
- 사용자가 안 푼 문제를 "이것도 풀어야 해요" 리스트로 쌓기
- 문제 풀이 당시의 Claude 힌트를 또 반복 설명하기
