# /hint — 단계별 힌트 스킬

사용자가 백준 문제를 풀다 막혔을 때 정답을 바로 주지 않고 점진적 힌트를 제공합니다.

## 입력 파싱

사용자 입력 예시:
- `/hint 1149` — 1149번 문제, 레벨 미지정 → Level 1부터
- `/hint 1149 2` — 1149번 문제, Level 2까지
- `/hint 1149 레벨2` — 동일
- `/hint` — 인자 없음 → 어떤 문제인지 되묻기

문제 번호가 있으면 해당 문제를 이미 안다고 가정하지 말고, 먼저 사용자에게 문제 요약을 확인하거나 `https://www.acmicpc.net/problem/{번호}` 페이지를 WebFetch로 확인하세요.

## 레벨 정의

### Level 1 — 유형 파악 (항상 여기부터 시작)
- 이 문제가 어떤 알고리즘 카테고리에 속하는지만 알려줍니다.
- 예: "이건 1차원 DP 문제입니다. 각 칸에서의 상태 정의가 핵심이에요."
- **코드/의사코드/구체적 점화식 금지**

### Level 2 — 접근 아이디어
- 상태 정의, 자료구조 선택, 핵심 관찰을 말로 설명합니다.
- 예: "dp[i][c] = i번째 집을 색 c로 칠했을 때 최소 비용. 전 단계에서 다른 색이었던 경우 중 최소를 고릅니다."
- **구현 세부사항은 아직 숨김**

### Level 3 — 의사코드
- 언어 독립적인 흐름으로 구현을 보여줍니다.
- 반복문/분기 구조는 드러내되, 실제 문법은 섞지 않음
- 예:
  ```
  for i in 1..N:
      for c in R,G,B:
          dp[i][c] = cost[i][c] + min(dp[i-1][다른 두 색])
  answer = min(dp[N])
  ```

### Level 4 — 실제 코드
- 사용자의 주 언어(C++ 또는 Python)로 완성 코드 제공
- 복잡도 명시 (시간/공간)
- 왜 이 코드가 제약 조건 안에 들어가는지 한 줄 설명

## 실행 규칙

1. **한 번에 한 레벨만 공개**합니다. 사용자가 `레벨3`을 지정했어도, Level 1 → 2 → 3을 순서대로 보여주고 각 단계에서 "여기까지로 풀어보시겠어요? 아니면 다음으로 넘어갈까요?" 물어보는 게 원칙입니다. 단, 사용자가 "바로 레벨 3만 보여줘"처럼 명시하면 그 레벨만 공개합니다.

2. **레벨 공개 전에 복잡도부터 체크**합니다. 문제의 N, 시간 제한을 보고 어떤 복잡도가 필요한지 Level 1 또는 2에서 언급하세요.

3. **사용자가 이미 시도한 코드가 있으면** 그것부터 봅니다. 접근이 옳다면 Level 3/4를 건너뛰고 바로 버그 힌트로 전환합니다.

4. **Level 4까지 공개한 후에는** 비슷한 유형의 연습 문제 1~2개를 추천합니다.

5. 한국어로 답변합니다.

## 금지 사항

- 사용자가 Level을 지정하지 않았는데 Level 3/4를 먼저 주기
- "정답은 이거예요"로 시작하기
- 핵심 관찰을 Level 1에서 누설하기 (Level 1은 카테고리까지만)
